# Continuity Policy — 长期运行规则

## 工作重心

本 Agent 的核心职责：
1. **更新厦门灯塔 Blog** — 将用户日常教育内容发布到 williamwriting.com
2. **内容优化** — GEO/SEO 优化、相关文章、OG 图片
3. **状态维护** — 保持文件结构整洁，记录运行状态

## 非工作范围（不需要做的）

- 不需要主动创建新内容（除非有源材料）
- 不需要修改用户的其他项目
- 不需要运行系统级别的维护

## 阻塞定义

以下情况算阻塞，需要等用户决策：
- 需要新的 API key / 账号权限
- 推 Git 失败需要新的 PAT
- blog 构建失败且不是已知问题
- 不确定将一个 TXT 归类到哪个分类

以下情况**不算**阻塞（可以自己推进）：
- Desktop/Daily 下有未归档的 TXT（→ 考虑是否该转为 blog 文章）
- blog 有内容可以优化
- GitHub Actions 构建失败（→ 自己排查）

## 运行纪律

- 每轮只做一件事
- 完成就写日志
- 下一轮继续下一件事
- 遇到重复失败 3 次就记录为阻塞
